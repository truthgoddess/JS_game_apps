Basic Math and Logic for Programming

Not all math classes teach the basic ideas or skills necessary to make programming easy to pick up. Algebra may have been a long time ago (or not yet!) or we don't connect basic patterns of speech or thinking to the programming that we do.

Well, it's time to change all that!

Computers are Stupid Tell Them Everything

There's nothing in this world as stupid as a computer. We don't normally think about computers that way, so when we learn programming, we can be shocked to find that our programs aren't doing what we want them to do. There's nothing on this planet that can do so much, but requires hand holding to get to each step of the way. Computers are like a toddler you have to teach to walk every step, think every thought, do every task.

So, first things first. Get ready to have to tell your program EVERYTHING. If you there is one thought that your program hasn't thought, you probably have to write it.

Thought Parking Variables

If you know algebra, then much of programming will make sense to you. If you don't know it, or you've forgotten it, a couple of ideas can help you get ready.

Imagine for a moment that you want to remember an important birthday. You write that birthday on a green sticky note and stick it to a mirror. You call that note "my green reminder" and the words on "my green reminder" are "Regina's birthday - 12/15!!!!!".

Later, while having dinner with your friend at home, you find that you have forgotten Regina's birthday again! You ask your friend if she remembers, and she says she doesn't, but wants to know. You tell her that location ("mirror"), the kind of thing to look for ("sticky note"), and ask her to go and get the birthday. She does, and yells from the mirror, "December 15th!"

This is pretty much the same thing as this:

var greensticky = "Regina's birthday - 12/15!!!";
var friendreminder = greensticky;

That is Javascript, and it is not dissimilar to many types of computer code you might encounter. In this case, the location where the greensticky is placed is not on a mirror, but in a specific piece of computer memory. Instead of there being an actual piece of green sticky paper, the piece of memory where the information will be stored will be called "greensticky." But "greensticky" contains the same text that our previous green sticky did: "Regina's birthday - 12/15!!!" So it's like we are writing our note on a bit of computer memory instead of paper.

We do this for the same reason we put things on sticky notes: to remember it later. That's what the second line of code is all about. It is our friend, going to the location of our green sticky note and getting the information about Regina's birthday to bring back to us. What do you think the code would look like if our virtual friend gave the information she found back to us? Did you guess this?

var usTheProgrammer = friendreminder;

Or something like it?

Directions for Later Functions

Let's say that you are housesitting for a friend, and your friend has left you a note:

  • There is plenty of food in the fridge. If you are hungry, eat that first!
  • The cats get fed at 8am and 6pm. Give them half a can (with water) each at those times.
  • If the AC starts clicking, either watch the cats hunt it (if you are in a good mood), or tap it lightly and it will stop.
  • If my neighbor calls with the tracking id of the package they sent to me where I am, can you call me and tell me what it is?
  • If I get any mail that seems important, just send a photo of it to me by text, ok?

Consider two things: the first is the note right after your friend wrote it; the second, when you read it for the first time.

First, when your friend first finishes that note, he leaves it somewhere and tells you where to find it. It contains instructions for what you should do, but isn't doing anything because you aren't there, haven't seen it, and don't know what it says.

Second, when you see the note for the first time, you get some information. You now know a little bit about what you are supposed to do in what circumstances. You don't immediately feed the cats, or tap the AC or start sending photos of mail. You simply have knowledge about the instructions that are there and when to start doing them. Later, when the conditions are right, you'll follow the instructions, but only then. Instructions are just instructions. They don't do anything until the situation arises in which the instructions need to be followed.

Functions in programming are like that. They are instructions written for later. They may look like they are active, running, moving pieces of code, but they aren't. They are there for any part of a program that needs to use that, and are only followed when the conditions are right.

function feedCats{
   //insert code here
}

function checkmail{
   //insert code here
}

The details of what the code instructions are up to you! But functions are not active until they are called:

feedCats();

That small bit of code is an example of something that tells the program to actually feed the cats! For a while it was just instructions, waiting, but typing something like that in your code is going to be like you actually looking at the instructions, getting the food from the refrigerator, putting water in it, and giving it to the cats. For a long time those instructions didn't do anything, but then at 8am they did!

So Little Done So Much Time

Because our devices run programs so fast that we can't see it, we come to expect that programming those devices is that simple, and takes as little time. Many of us when we first learn programming can be shocked at how long it takes to make such fast moving simple useful programs. It is quite amazing what they can do, but be patient!

Good luck, and have fun!
